What is grid layout?

A grid is a collection of horizontal and vertical lines creating a pattern against which we can line up our design elements. They help us create layouts in which our elements won't move around or change width as we move from page to page, providing greater consistency on our websites.

A grid will typically have columns, rows, and gutters, or gaps between each row and column.

Defining a grid

Reference the sample grid HTML here and your local copy of the same markup here.

Similar to how you define a flexbox, you define a grid layout by setting the value of the display property to grid using the following CSS:

Input:

CSS
.grid-container-1 {
  display: grid;
}

HTML
<div class="grid-container">
  <div>One</div>
  <div>Two</div>
  <div>Three</div>
  <div>Four</div>
  <div>Five</div>
  <div>Six</div>
  <div>Seven</div>
</div>

Note: it was an intentional choice not to apply styling to the class grid-container.

Unlike flexbox, the items will not look immediately different. Declaring display: grid gives you a one column grid, so your items will continue to display one below the other as they do in normal flow. To see something that looks more grid-like, we'll need to add some columns to the grid. Let's add three 200-pixel columns. You can use any length unit or percentage to create these column tracks.

Input:

CSS
.grid-container-1 {
  display: grid;
  grid-template-columns: 200px 200px 200px;
}

Flexible grids with the fr unit

In addition to creating grids using lengths and percentages, we can use the fr unit. The fr unit represents one fraction of the available space in the grid container to flexibly size grid rows and columns. Change your track listing to the following definition, creating three 1fr tracks:

Input:

CSS
.grid-container-2 {
  display: grid;
  grid-template-columns: 1fr 1fr 1fr;
}

HTML
<div class="grid-container-2">
  <div class="grid-color-1">One</div>
  <div class="grid-color-1">Two</div>
  <div class="grid-color-1">Three</div>
  <div class="grid-color-1">Four</div>
  <div class="grid-color-1">Five</div>

You'll notice that the first track gets 2fr of the available space and the other two tracks get 1fr, making the first track larger. You can mix fr units with fixed length units. In this case, the space needed for the fixed tracks is used up first before the remaining space is distributed to the other tracks.

Note: The fr unit distributes available space, not all space. Therefore, if one of your tracks has something large inside it, there will be less free space to share.

Gaps between tracks

To create gaps between tracks, we use the following properties:

  • column-gap for gaps between columns
  • row-gap for gaps between rows
  • gap as shorthand for both

Input:

CSS
.grid-container-4 {
  display: grid;
  grid-template-columns: 2fr 1fr 1fr
  gap: 20px;
}

<div class="grid-container-4">

Repeating track listings

You can repeat all or merely a section of your track listing using the CSS repeat() function. Change your track listing to the following:

Input:

CSS
  .grid-container-5 {
    display: grid;
    grid-template-columns: repeat(3, 1fr);
    gap: 20px;
}

Implicit and explicit grids

Up to this point, we've specified only column tracks, but rows are automatically created to hold the content. This concept highlights the distinction between explicit and implict grids:

  • Explicit grid: created using grid-template-columns or grid-template-rows.
  • Implicit grid: extends the defined explicit grid when content is placed outside of that grid, such as into the rows by drawing additional grid lines.

By default, tracks created in the implicit grid are auto sized, which generally means that they're large enough to contain their content. If you wish to give implicit grid tracks a size, you can use the grid-auto-rows and grid-auto-columns properties. If you add grid-auto-rows with a value of 100px to your CSS, you'll see that the rows are now 100px tall.

Input:

CSS
  .grid-container-6 {
    display: grid;
    grid-template-columns: repeat(3, 1fr);
    grid-auto-rows: 100px;
    gap: 20px;
}

  <div class="grid-color-1">Five</div>

The minmax() function

Our 100px tall tracks won't be very useful if we add content into those tracks that's taller than 100px, in which case it would cause overflow. It might be a better idea to have tracks that are at least 100px tall and can still expand if more content becomes added.

Input:

CSS
  .grid-container-7 {
    display: grid;
    grid-template-columns: repeat(3, 1fr);
    grid-auto-rows: minmax(100px, auto);
    gap: 20px;
}
      
HTML
<div class="grid-container-7">
  <div class="grid-color-1">One</div>
  <div class="grid-color-1">Two</div>
  <div class="grid-color-1">Three</div>
  <div class="grid-color-1">Four</div>
  <div class="grid-color-1">Five</div>
  <div class="grid-color-1">Six</div>
  <div class="grid-color-1">Seven</div>
</div>

As many columns as will fit

We can combine some of the lessons we've learned about track listing, repeat notation, and minmax() to create a useful pattern. Sometimes it's helpful to be able to ask a grid to create as many columns as will fit into the container. We do this by setting to value of grid-template-columns using the repeat() function, but instead of entering a number, we use the value auto-fit. For the second parameter of the function, we use minmax() with a minimum value equal to the minimum track size that we would like to have and a maximum of 1fr.

Input:

CSS
  .grid-container-8 {
    display: grid;
    grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));
    grid-auto-rows: minmax(100px, auto);
    gap: 20px;
}
      
HTML
<div class="grid-container-8">
  <div class="grid-color-1">One</div>
  <div class="grid-color-1">Two</div>
  <div class="grid-color-1">Three</div>
  <div class="grid-color-1">Four</div>
  <div class="grid-color-1">Five</div>
  <div class="grid-color-1">Six</div>
  <div class="grid-color-1">Seven</div>
</div>

Output:

One
Two
Three
Four
Five
Six
Seven

This works because the grid is creating as many 200px columns as will fit into the container, then sharing whatever space if leftover among all the columns. The maximum is 1fr, which, as we already know, distributes space evenly between tracks (since this is how the fr unit naturally behaves).

Line-based placement

We'll now take a look at how to place things on a grid. Our grid always has lines--these are numbered beginning with 1 and relate to the writing mode of the document. Column line 1 in English, for instance, would be on the left-hand side of the grid in accordance with how English text is written from left-to-right (with row 1 at the top), while column line 1 would be on the right hand side in Arabic.

To position items along these lines, we can specify the start and end lines of the grid area where an item should be placed using the following properties:

  • grid-column-start
  • grid-column-end
  • grid-row-start
  • grid-row-end

These properties accept line numbers as their values; this means we can specify that an item should start on line 1 and end on line 3. Alternatively, you can also use shorthand properties that let you specify the start and end lines simultaneously, separated by a forward slash /:

  • grid-column: shorthand for grid-column-start and grid-column-end
  • grid-row: shorthand for grid-row-start and grid-row-end

You can see an example of this in action here, a local copy of which is available here. Note that this sample has a defined grid and a simple article outlined. You can see that auto-placement is placing each item into its own cell in the grid.

Let's arrange the elements for our site by using the grid lines. Add the following rules to the bottom of your CSS on the sample page to see how if affects the layout:

CSS
header {
  grid-column: 1 / 3;
  grid-row: 1;
}

article {
  grid-column: 2;
  grid-row: 2;
}

aside {
  grid-column: 1;
  grid-row: 2;
}

footer {
  grid-column: 1 / 3;
  grid-row: 3;
}

NOTE: You can also use the value -1 to target the end column or row line, then count inwards from the end using negative values. Note also that lines count always from the edges of the explicit grid and not the implicit-grid.

Positioning with grid-template-areas

An alternative way to arrange items on your grid is to use the grid-template-areas property and give the various elements of your design a name.

Comment out the line-based positioning from the last example on the grid lines placement sample page and add the following CSS:

CSS
.container {
  display: grid;
  grid-template-areas:
    "header header"
    "sidebar content"
    "footer footer";
  grid-template-columns: 1fr 3fr;
  gap: 20px;
}

header {
  grid-area: header;
}

article {
  grid-area: content;
}

aside {
  grid-area: sidebar;
}

footer {
  grid-area: footer;
}

After adding this to your CSS, reload the page, and you'll see that your items have been placed as before but without us needing to use any line numbers!

The rules for grid-template-areas are as follows:

  • You need to have every cell of the grid filled.
  • To span across two cells, repeat the name.
  • To leave a cell empty, use a . —aka a period.
  • Areas must be rectangular—for example, you can't have an L-shaped area.
  • Areas can't be repeated in different locations.

You can play around with our layout, changing the footer to only sit underneath the article and the sidebar to span all the way down. This is a very nice way to describe a layout because it's clear what's happening just from looking at the CSS.

Nesting grids and subgrid

It's possible to create a grid within a grid, or a subgrid. You can do this by setting the display: grid property on an item.

Let's expand on the previous example by adding a container for articles and using a nested grid to control the layout of multiple articles. While we're using only one column in the nested grid, we can define the rows to be split in a 2:1:1 ratio by using the grid-template-rows property. This approach allows us to create a layout where one article at the top of the page has a large display, while the others have a smaller, preview-like layout.

Add the following CSS to your line-based placement sample page:

CSS
.articles {
  display: grid;
  grid-template-rows: 2fr 1fr 1fr;
  gap: inherit;
}

article {
  padding: 10px;
  border: 2px solid rgb(79 185 227);
  border-radius: 5px;
}

To make it easier to work with layouts in nested grids, you can use subgrid on grid-template-rows and grid-template-columns properties. This allows you to leverage the tracks defined in the parent grid.

In the following example, we're using line-based placement, enabling the nested grid to span multiple columns and rows of the parent grid. We've added a subgrid to inherit the parent grid's column tracks while adding a different layout for the rows within the nested grid.

Input:

CSS
.grid-container-9 div {
  border-radius: 5px;
  padding: 10px;
  background-color: rgb(207 232 220);
  border: 2px solid rgb(79 185 227);
}
  
.grid-container-9 {
  display: grid;
  grid-template-columns: repeat(4, 1fr);
  grid-template-rows: repeat(1, 1fr);
  gap: 10px;
}

#subgrid {
  grid-column: 1 / 4;
  grid-row: 2 / 4;
  display: grid;
  gap: inherit;
  grid-template-columns: subgrid;
  grid-template-rows: 2fr 1fr;
}

HTML
<div class="grid-container-9">
  <div>One</div>
  <div>Two</div>
  <div>Three</div>
  <div>Four</div>
  <div id="subgrid">
    <div>Five</div>
    <div>Six</div>
    <div>Seven</div>
    <div>Eight</div>
  </div>
  <div>Nine</div>
  <div>Ten</div>
</div>

Grid frameworks

Numerous grid frameworks are available, offering a 12- or 16-column grid, to help with laying out your content. The good news is you probably won't need any third-party frameworks to help you create grid-based layouts—grid functionality is already included in the specification and supported by most modern browsers.

Add the following CSS to your grid frameworks sample page and see how it affects the layout:

CSS
header {
  grid-column: 1 / 13;
  grid-row: 1;
}

article {
  grid-column: 4 / 13;
  grid-row: 2;
}

aside {
  grid-column: 1 / 4;
  grid-row: 2;
}

footer {
  grid-column: 1 / 13;
  grid-row: 3;
}
