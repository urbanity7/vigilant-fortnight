Normal flow

Normal flow is the standard layout that's used on a page barring other inputs to control the page layout. It displays content in the order in which it appears in the HTML.

In normal flow, the elements appear one below the other (e.g., this paragraph is directly below the previous paragraph) as block elements. In comparison, inline elements are used when blocks appear next to one another.

Finally, the below elements can be used to change how content are laid out in CSS:

  • The display property -- standard values such as block, inline, or inline-block can change how elements behave in normal flow, for example, by making a block-level element behave like an inline element (see Types of CSS Boxes for more info/review). Some layout methods, like CSS Grid and Flexbox, are enabled using display values and alter how child elements are laid out inside their parent elements.
  • Floats -- applying a float value such as left can cause block-level elements to wrap along one side of an element, like the way images sometimes have text floating around them in magazine layouts.
  • The position property -- allows you to precisely control the placement of boxes inside other boxes. static positioning is default in normal flow, but you can cause other elements to be laid out differently using other values, for example, as fixed to the top of the browser viewport.
  • Table layout -- features designed for styling parts of an HTML table can be used on non-table elements using display: table and associated properties.
  • Multi-column layout -- the multi-column layout properties can cause the content of a block to layout in columns, as you might see in a newspaper.

The display property

The main methods for achieving page layout in CSS all involve specifying values for the display property. This property allows us to change the default way something displays. Everything in normal flow has a default value for display; i.e., a default way that elements are set to behave. For instance, <p> elements use display: block when written in English, while <a> elements use display: inline automatically.

You can change this default display behavior. For example, the <li> element is display: block by default, meaning that list items display one below the other in our document as it's written in English. If we were to change the display value to inline, these items would display next to each other similarly to how works appear next to each other in a sentence. The fact that you can change the value of display for any element means that you can pick HTML elements for their semantic meaning without being concerned about how they will look. In other words, the way they look is something that you can change.

In addition to being able to change the default presentation by turning an item from block to inline and vice versa, there are some more involved layout methods that start out as a value of display. However, when using these you will generally need to invoke additional properties. The two values most important for our discussion of layout are display: flex and display: grid.

Flexbox

To use the flexbox layout, apply the display: flex property to the parent element of the elements you want to appear in the flexbox layout; all of this element's direct children will then become flex items.

Setting display: flex

The HTML markup below gives us a containing element with a class of wrapper-flex-1, inside of which are three <div> elements. By default, these would display as block elements.

However, if we add display: flex to the parent, the three items now arrange themselves into columns. This is because they therefore become flex items and are being affected by some initial values that flexbox sets on the flex container. They are displayed in a row because the property flex-direction of the parent element has an initial value of row. They all appear to stretch in height because the property align-items of their parent element has an initial value of stretch. This means that the items stretch to the height of the flex container, which in this case is defined by the tallest item. The items all line up at the start of the container, leaving any extra space at the end of the row.

Input:

CSS
.wrapper-flex-1 {
  display: flex;
}

.box-flex-1, 
.box-flex-2, 
.box-flex-3 {
    background-color: lightblue;
    padding: 5px;
    border-radius: 5px;
}

HTML
<div class="wrapper">
  <div class="box-flex-1">One</div>
  <div class="box-flex-2">Two</div>
  <div class="box-flex-3">Three</div>
</div>

Output:

One
Two
Three

Setting the flex property

In addition to properties that can be applied to a flex container, there are also properties that can be applied to flex items. These properties, among other things, can change the way that items flex, enabling them to expand or contract according to available space.

As a simple example, we can add the flex property to all of our child items, and give it a value of 1. This will cause all of the items to grow and fill the container, rather than leaving space at the end. If there is more space then the items will become wider; if there is less space they will become narrower. In addition, if you add another element to the markup, the other items will all become smaller to make space for it; the items all together continue taking up all the space.

Input:

CSS
.wrapper-flex-1 {
  display: flex;
}

.wrapper-flex-1 > 
.box-flex-1-option2, 
.box-flex-2-option2, 
.box-flex-3-option2 {
  flex: 1;
}

.box-flex-1-option2, 
.box-flex-2-option2, 
.box-flex-3-option2 {
  background-color: lightblue;
  padding: 5px;
  border-radius: 5px;
}
      
HTML
<div class="wrapper">
  <div class="box-flex-1-option2">One</div>
  <div class="box-flex-2-option2">Two</div>
  <div class="box-flex-3-option2">Three</div>
</div>

Output:

One
Two
Three

Grid layout

Unlike flexbox, which is designed for one-dimensional layouts, grid layout is designed for two-dimensional layouts--or those that require things to line up in both rows and columns rather than one or the other.

Setting display: grid

Like flexbox, we enable the grid layout using the display property. More specifically, we use display: grid. Additionally, we also need to define row and column tracks for the parent using grid-template-rows and grid-template-columns.

In the example below, we've defined three columns of 1fr each, as well as two rows of 100px. We don't need to put any rules on our child elements; they're automatically placed into the cells our grid's created.

Input:

CSS
.wrapper-grid {
  display: grid;
  grid-template-columns: 1fr 1fr 1fr;
  grid-template-rows: 100px 100px;
  gap: 10px;
}

.box-grid-1,
.box-grid-2,
.box-grid-3,
.box-grid-4,
.box-grid-5,
.box-grid-6 {
    background-color: lightblue;
    padding: 5px;
    border-radius: 5px;
}
  
HTML
<div class="wrapper">
  <div class="box-grid-1">One</div>
  <div class="box-grid-2">Two</div>
  <div class="box-grid-3">Three</div>
  <div class="box-grid-4">Four</div>
  <div class="box-grid-5">Five</div>
  <div class="box-grid-6">Six</div>
</div>

Output:

One
Two
Three
Four
Five
Six

Placing items on the grid

Once you have a grid, you can explicitly place your items on it, rather than relying on the auto-placement behavior seen above. In the example below, we've defined the same grid, but this time with three child items. We've set the start and end line of each item using the grid-column and grid-row properties. These properties cause the items to span multiple tracks.

Input:

CSS
.wrapper-grid {
  display: grid;
  grid-template-columns: 1fr 1fr 1fr;
  grid-template-rows: 100px 100px;
  gap: 10px;
}

.box-grid-1-2 {
  grid-column: 2 / 4;
  grid-row: 1;
  background-color: lightblue;
  padding: 5px;
  border-radius: 5px;
}

.box-grid-2-2 {
  grid-column: 1;
  grid-row: 1 / 3;
  background-color: lightblue;
  padding: 5px;
  border-radius: 5px;
}

.box-grid-3-2 {
  grid-column: 3;
  grid-row: 2;
  background-color: lightblue;
  padding: 5px;
  border-radius: 5px;
}

HTML
<div class="wrapper">
  <div class="box-grid-1-2">One</div>
  <div class="box-grid-2-2">Two</div>
  <div class="box-grid-3-2">Three</div>
</div>

Output:

One
Two
Three
