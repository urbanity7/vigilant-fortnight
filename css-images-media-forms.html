Replaced elements

Images and videos are replaced elements, or elements of which CSS cannot affect their internal layout--instead, CSS can only affect their position on the page among other elements.

Some replaced elements, including images and video, also have an aspect ratio. Items with an aspect ratio have a size in both the horizontal (x) and vertical (y) directions, and will be displayed using their intrinsic dimensions by default.

Sizing images

Because everything in CSS generates a box, placing an image inside a box that is smaller or larger than the intrinsic dimensions of the image file in either direction will cause the image to appear smaller or larger (i.e., overflow) the box. Thus, you need to make a decision on what happens with the overflow.

In the example below, we have two boxes, both 200px in size:

  • One contains an image which is smaller than 200px--it is smaller than the box and thus doesn't stretch to fill it.
  • The other image is larger than 200px and overflows the box.

To fix the overflow issue, we can set the image's max-width property to 100%. This will enable the image to become smaller in size than the box, but not larger. This technique also works with other replaced elements, including <video> and <iframe>.

You can also make other choices about images inside containers. For example, you may want to size an image so it completely covers a box. In such an instance, try using the object-fit property. This property enables you to size replaced elements to fit a box in a variety of ways.

In the below example, we have used the value cover, which sizes the image down while maintaining its aspect ratio so that it neatly fills the box. As the aspect ratio is maintained, some parts of the image will be cropped by the box.

If we use contain as a value, the image will be scaled down until it is small enough to fit inside the box, resulting in "letterboxing" if the image is not the same aspect ratio as the box.

Finally, the fill value will fill the box with the image, but will not maintain the aspect ratio.

Replaced elements in layout

When using various CSS layout techniques on replaced elements, you may sometimes find that they behave slightly differently from other elements. For example, in a flex or grid layout, elements are stretched to fill the entire area by default. However, images will not stretch, and will instead be aligned to the start of the grid area or flex container.

You can see this happening in the example below where we have a two column, two row grid container, which has four items in it. All of the <div> elements have a background color and stretch to fill the row and column. The image, however, does not stretch.

If you are following these lessons in order, then you may have not looked at layout yet. Just keep in mind that replaced elements, when they become part of a grid or flex layout, have different default behaviors, essentially to avoid them being stretched strangely by the layout.

To force the image to stretch to fill the grid cell it is in, you'd have to do something like the following:

img {
  width: 100%;
  height: 100%;
}

This would, however, stretch the image, so it is probably not what you'd want to do.

Form elements

Styling form elements with CSS can be tricky. The Web Forms module has detailed guides to the more complex aspects of styling these, so they're not fully covered in this section. However, there are some basics worth being aware of.

Many form controls are added to the page using the <input> element. This element defines simple form fields, such as text inputs, as well as more complex fields, like date and color pickers. There are some additional elements, such as <textarea> for multiline input, as well as elements used to contain and label parts of forms such as <fieldset> and <legend>.

HTML also contains attributes that enable web developers to indicate which fields are required, and even the type of content that needs to be entered. If the user enters something unexpected, or leaves a required field blank, the browser can leave an error message. Different browsers vary with one another in how much styling and customization they allow for such items.

Styling text input elements

Elements that allow for text input, such as <input type="text">, and the more specific <input type="email">, and the <textarea> element are quite easy to style and behave just like other boxes on your page. The default styling of these elements will differ, however, based on the operating system and browser that your user visits the site with.

In the example below, we have styled some text inputs using CSS--you can see that things like borders, margins, and padding all apply like you'd expect. We are using attribute selectors to target the different input types. Be careful when changing the styling of form elements in order to ensure it's obvious to users that they're form elements.

As the lessons on styling web forms indicate, many of the more complex input types are rendered by the operating system and are therefore inaccessible to styling. For this reason, you should always text complex forms in multiple browsers as well as assume that forms are going to vary in appearance for different visitors.

Inheritance and form elements

In some browsers, form elements do not inherit font styling by default. Therefore, if you want to be sure that your form fields use the font defined on the body, or that on a parent element, you should add this rule to your CSS:

CSS
button, input, select, textarea {
  font-family: inherit;
  font-size: 100%;
}

Form elements and box-sizing

Across browsers, form elements use different box sizing rules for different widgets. You learned about the box-sizing property in the box model lesson, and you can use this knowledge when styling forms to ensure a consistent experience when setting widths and heights on form elements.

For consistency, it's a good idea to set margins and padding to 0 on all elements, and then go back and add these back in when styling particular controls:

CSS
button, input, select, textarea {
  box-sizing: border-box;
  padding: 0;
  margin: 0;
}

Other useful settings

In addition to the rules mentioned above, you should also set overflow: auto on <textarea> objects to stop the browser from showing a scrollbar when there's no need for one:

CSS
textarea {
  overflow: auto;
}

Putting it all together into a "reset"

As a final step, we can wrap up the various properties discussed above into the following "form reset" to provide a consistent base to work from. This includes all the items mentioned in the last three sections:

CSS
button, input, select, textarea {
  font-family: inherit;
  font-size: 100%;
  box-sizing: border-box;
  padding: 0;
  margin: 0;
}

textarea {
  overflow: auto;
}

Note: Normalizing stylesheets are used by many developers to create a set of baseline styles to use on all projects. Typically these do similar things to those described above, making sure that anything different across browsers is set to a consistent default before you do your own work on the CSS. They are not as important as they once were, as browsers are typically more consistent than in the past. However, if you want to take a look at one example, check out Normalize.css, which is a very popular stylesheet used as a base by many projects.
