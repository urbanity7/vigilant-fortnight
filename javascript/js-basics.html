Internal vs external JavaScript

Internal

  • Enclosed in <script> tags within the <head> or <body>

External

  • Linked to similarly to CSS style sheets, except the opening tag is <script>, not <link>
  • <script src="name-of-js-file.js"></script>
    • Advantages
    • Separates HTML and JavaScript, making both easier to read and maintain
    • Cached JavaScript files can make pages load faster

Overview of JavaScript syntax

There are two types of values:

Fixed values, or literals

  • Strings are text enclosed in double or single quotes
  • "John Doe", 'John Doe'

Variable values, or variables

  • Variables are used to store data values
  • Signified via one of the following keywords and then defined by a value with an = sign:
    • var
      • Can be scoped globally or locally
        • Globally scoped vars are declared outside of a function. They can be called anywhere in the window--not just within a specific function
        • Locally scoped vars are declared inside a function, and thus must be called within the same function
      • Can be hoisted, or moved to the top of their scope before the code is run. They are initialized with a value of undefined
        • In other words, var can be declared after the first instance in which it appears in your code
      • Both types can be updated and redeclared within their respective scopes
    • let
      • Block scoped (aka anything within a set of curly brackets), meaning it can't be used outside the block in which it is declared
      • Can be updated within its scope, but not redeclared
        • The following would work:
        • let greeting = "Say hi";
greeting = "say hello instead";
        • The following would not work (the second use of let is what redeclares the variable):
        • let greeting = say hi;
let greeting = "say hello instead";
        • It's possible to define the same variable in different scopes, meaning the following would also work because the code treats each instance of the word "let" as a different scope due to the use of curly brackets: 
        • let greeting = say hi;
if (true) { 
  let greeting = "say hello instead";
  console.log(greeting); // "say hello instead"
}
console.log(greeting); // "say hi"
      • Can be hoisted to the top, but are not initialized when this is done
    • const
      • const values are constant values
      • Block scoped
      • Cannot be updated or redeclared--the whole point of const values is that they remain constant. This means that every const declaration must be initialized when it's first declared
        • That said, you can update the properties of const variables. For instance, the following would work:
        • greeting.message = "say hello instead";
        • However, this would not work because it's updating the value stored within const rather than a property of const:
        • greeting = {
  words: "Hello",
  number: "five",
} // error: Assignment to constant variable.
      • Can be hoisted to the top, but are not initialized when this is done
  • Variables can be defined or undefined
    • Undefined variables end after the variable is identified:
    • var/let/const x;

    • Defined variables use an assignment operator aka an = sign:
    • var/let/const x;
x = 6;

  • When to use var vs let vs const
    • let and const are preferred over var
      • const should be used when the value and the property type (arrays, objects) should not be changed; try to use let otherwise
    • var should be used in cases where you MUST support older browsers

Comments in JavaScript

Not all JavaScript statements are executed when the code is run. Code after double slashes // or between /* and */ is treated as a comment and not code to be run--think comments in CSS

  • Note: Comments that begin with // do not need a // at the end of the comment because they are single-line comments--the comment ends when a new line starts
  • /* */ comments are reserved for comments that span multiple lines

JavaScript names/identifiers

Names/identifiers are used to name variables, keywords, and functions. They must adhere to the following rules:

  • Begin with a letter, dollar sign, or underscore--names/identifiers cannot start with numbers
  • Subsequent characters may be letters, digits, underscores, and dollar signs

Capitalization/punctuation rules

  • JavaScript is case sensitive--the variables "lastname" and "lastName" are distinct
  • JavaScript is typically written using Camel Case (e.g., camelCase)
    • This is because JavaScript only permits hyphens for subtracting and negative numbers--they can't be used when naming names/identifiers
