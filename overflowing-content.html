What is overflow?

Overflow happens when there is too much content to fit in a box. Remember, everything in CSS is a box. Because CSS tries to avoid "data loss," it has a tendency to permit overflow unless otherwise instructed not to.

The overflow property

The overflow property is used to control overflow. It is set to visible by default, which allows overflow to be seen on the frontend. It can also be set to the following:

  • To crop content when it overflows, you can use overflow: hidden. Because this makes overflow invisible, only use this when you're sure hiding content won't cause problems.
  • overflow: scroll adds scrollbars when content overflows to keep the layout consistent while showing all pertinent content. To scroll on just one axis instead of both, you can use the corresponding property for that axis (e.g., overflow-y: scroll).
    • It is not recommended to use overflow-x: scroll to accommodate long words; use word-break or overflow-wrap, or CSS sizing methods, instead.
    • If you only want scrollbars to appear when there is more content than can fit in the box, use overflow: auto to allow the browser to determine if it should display scrollbars.

Overflow establishes a Block Formatting Context

When using the values scroll and auto with the overflow property, you create a Block Formatting Context (BFC). This essentially means that the content of an element box with these overflow values acquires a self-contained layout; content outside, like an element box, cannot poke into the element box, and nothing from the element box can poke into the surrounding layout. This in turn enables scrolling behavior, as all box content needs to be contained and not overlap to create a consistent scrolling experience.

Unwanted overflow in web design

Unlike older CSS layout methods, newer methods are designed with an eye to managing overflow. Unlike their predecessors, then, they largely work without assumptions or dependencies for how much content there will be on a web page. Keep this in mind when working with older CSS layouts, as you'll likely need to accommodate overflow in these instances.
