Block and inline boxes

There are two overarching types of boxes: block boxes and inline boxes. The general box type dictates how the box behaves and flows relative to other content on the page. Boxes also have both an inner and outer display type.

Outer display type

For boxes with an outer display type of block:

  • The box will break onto a new line.
  • The width and height properties are respected.
  • Padding, margin, and border will cause other elements to be pushed away from the box.
  • If width is not specified, the box will extend in the inline direction to fill the space available in its container. In most cases, the box will become as wide as its container.
  • Keep in mind that the following elements use an outer display type of block by default:
    • <h1>
    • <p>

For boxes with an outer display type of inline:

  • The box will not break onto a new line.
  • The width and height properties will not apply.
  • Top and bottom padding, margins, and borders will apply, but will not cause other inline boxes to move away from the box.
  • Left and right padding, margins, and borders will apply and will cause other inline boxes to move away from the box.
  • Keep in mind that the following elements use an outer display type of inline by default:
    • <a>
    • <span>
    • <em>
    • <strong>

Inner display type

The inner display type determines how elements inside the box are ordered. It is the default way things behave on the web--and, barring additional instructions, the elements inside a box are likewise laid out in normal flow and behave as block or inline boxes.

You can change the inner display type using the CSS display property (e.g., display: flex). Any direct children of this box will likewise adopt this inner display type (in this case, flex). Defining an inline display type does not change the box's outer display type.

Controlling display types

***To change a box's inner display type, use the following CSS:

display: type;

***To change a box's outer display type, use the following CSS:

display-outside: block/inline;

*** = clarify this is correct

To toggle between inline and block display modes, you can change display: inline to display: block or display: inline-flex to display: flex.

What is the CSS box model?

The box model uses block boxes; only some aspects of inline boxes follow the behavior of the box model. There are also two different box models: the standard model--which is what browsers use by default--and the alternate box model.

CSS parts of a box:

  • Content box: the area where your content is displayed; size it using properties like inline-size and block-size or width and height.
  • Padding box: the padding sits around the content as white space; size it using padding and related properties.
  • Border box: the border box wraps the content and any padding; size it using border and related properties.
  • Margin box: the margin is the outermost layer, wrapping the content, padding, and border as whitespace between this box and other elements; size it using margin and related properties.

The standard CSS box model

In the standard box model, if you give a box an inline-size and a block-size (or width and height) attributes, this defines the inline-size and block-size (or width and height) of the content box. Any padding and border is then added to those dimensions to get the total size taken up by the box.

So, if we assume a box has the following CSS,

.box {
  width: 350px;
  height: 150px;
  margin: 10px;
  padding: 25px;
  border: 5px solid black;
}

Then the actual space taken up by the box will be 410px wide (350 + 25 + 25 + 5 + 5) and 210px high (150 + 25 + 25 + 5 + 5).

Keep in mind, the margin is not counted towards the actual size of the box despite the fact it affects how much space the box takes up on the page (specifically the space outside the box). The box's area stops at the border--it does not extend into the margin.

The alternative CSS box model

In the alternative box model, any width is the width of the visible box on the page. The content area width is that width minus the width for the padding and border. You don't need to add up the border and padding to get the real size of the box.

To turn on the alternative box model, use the following CSS:

.box {
  box-sizing: border-box;
}

So, if we assume a box has the following CSS from above (additions denoted by italicized text),

.box {
  width: 350px;
  inline-size: 350px;
  height: 150px;
  block-size: 150px;
  margin: 10px;
  padding: 25px;
  border: 5px solid black;
}

Then the actual space taken up by the box will be 350px in the inline direction and 150px in the block direction.

To use the alternative box model for all of your elements (a common choice among developers), set the box-sizing property on the <html> element and set all other elements to inherit that value, as such:


html {
  box-sizing: border-box;
}  

*,
*::before,
*::after {
  box-sizing: inherit;
}

Margins, borders, and padding

Margins

The margin refers to the invisible space around your box that pushes elements away from the box itself.

Margins can have positive or negative values. Setting a negative margin on one side of your box can cause it to overlap with other things on the page.

Regardless of whether you're using the standard box model or the alternative box model, the margin is always added after the size of the visible box has been calculated.

Margin collapsing: Depending on whether two elements whose margins touch have positive or negative margins, the results will be different.

  • Two positive margins will combine into one margin with a size equal to the largest individual margin.
  • Two negative margins will collapse and the smallest value will be used.
  • If one margin is negative, its area will be subtracted from the total.

A good rule of thumb is that margin collapsing occurs when you end up with space in the margins that you didn't expect. See here for more details.

Borders

The border is drawn between the box's margin and padding. In the standard box model, the size of the border is added to the width and height of the content box; in the alternative box model, the size of the border makes the size of the content box smaller as it takes up some of its available width and height.

You can set the top/right/bottom/left borders individually using the same notation as when setting the properties of individual margins. You can also add properties like width and style to these individually-set borders through properties like border-top-width and border-left-style.

Padding

The padding sits between the border and the content area, and pushes contents away from the border.

Unlike margins, you cannot set the padding to a negative value. Any background applied to your element will always display behind the padding.

The box model and inline boxes

All of the above fully applies to block boxes. Some of these properties can also apply to inline boxes, such as those created using <span> elements, for example.

Input:

CSS
span {
  margin: 20px;
  padding: 20px;
  width: 80px;
  height: 50px;
  background-color: lightblue;
  border: 2px solid blue;
}

HTML
<p>I am a paragraph and this is a <span>span</span>
inside that paragraph. A span is an inline element and 
so does not respect width and height.<p>

Output:

I am a paragraph and this is a span inside that paragraph. A span is an inline element and so does not respect width and height.

Using display: inline-block

display: inline-block is a special value of display. It provides a middle ground between inline and block. Use it if you do not want an item to break onto a new line, but do want it to respect width and height without the overlapping in the previous input/output example.

If an element is set to display: inline-block:

  • The width and height properties are respected.
  • The padding, margin, and border will cause other elements to be pushed away from the box.
  • It will NOT break onto a new line.
  • It will only become larger than its content if you explicitly add width and height properties.

The next examples show a <span> element that has a display: inline-block property, display: block property, and lack thereof. There is no example with a display: inline property because <span> elements are inline by default.

    Example of a <span> element with a display: inline-block property:

    I am a paragraph and this is a span inside that paragraph. A span is an inline element and so does not respect width and height.

    Example of a <span> element with a display: block property:

    I am a paragraph and this is a span inside that paragraph. A span is an inline element and so does not respect width and height.

    Example of a <span> element with no display property:

    I am a paragraph and this is a span inside that paragraph. A span is an inline element and so does not respect width and height.

Where this can be useful is when you want to give a link a larger hit area by adding padding. The <a> element is an inline element by default like <span>; this means you can use display: inline-block to allow padding to be set on it, making it easier for a user to click the link.

You see this fairly frequently in navigation bars.
