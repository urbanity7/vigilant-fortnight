Writing Modes

Writing modes set the direction in which block-level elements are displayed on the page--left-to-right, top-to-bottom, etc.--and, consequently, the direction in which text flows in sentences. They are controlled using the writing-mode property, and accept the following values:

  • horizontal-tb: top-to-bottom block flow direction with sentences running horizontally.
  • vertical-rl: right-to-left block flow direction with sentences running vertically.
  • vertical-lr: left-to-right block flow direction with sentences running vertically.

Writing modes and block and inline layout

Setting a writing mode also affects block and inline layout. This is because when we switch the writing mode, we are changing which direction is block and which is inline. In a horizontal-tb writing mode, the block direction runs from top to bottom; in a vertical-rl writing mode, the block directions runs right-to-left horizontally. So, the block dimension is always the direction blocks are displayed on the page in the writing mode in use. Meanwhile, the inline dimension is always the direction a sentence flows. Direction

Because writing mode and the direction of text can change, newer CSS layout methods do not refer to left and right, and top and bottom. Instead, they use the terms start and end alongside the idea of inline and block.

Logical properties and values

CSS uses mapped properties--called logical or flow relative versions--in lieu of the width and height properties (which are physical properties) to control these dimensions when using a writing mode. These correspond to the following:

  • inline-size is used for width when in a horizontal writing mode. It refers to the size in the inline dimension.
  • block-size is used for height and is the size in the block dimension.

Local margin, border, and padding properties

The margin-top, padding-left, and border-bottom properties also have corresponding logical properties. These include:

  • The margin-top property is mapped to margin-block-start. This will always refer to the margin at the start of the block dimension.
  • The padding-left property maps to padding-inline-start, the padding that is applied to the start of the inline direction. This will be where sentences start in that writing mode.
  • The border-bottom property maps to border-block-end, which is the border at the end of the block dimension.

Keep in mind that physical properties stay tied to their physical direction, while logical properties switch with the writing mode. You can see this at play under the "Local margin, border, and padding properties" section on this page.

Logical values

There are also some properties that take physical values of top, right, bottom, and left. These values also have mappings, to logical values--block-start, inline-end, block-end, and inline-start.

As mentioned above, physical properties stay tied to their physical direction, while logical properties switch with the writing mode. You can see this at play under the "Logical values" section on this page.

Should you use physical or logical properties?

Logical properties are newer than their physical counterparts, so some browsers have only recently started supporting their use. While logical properties will likely become the standard in the future--because they make more sense in the context of layout methods such as flexbox and grid--it therefore might be preferable to use their physical counterparts when you're only using a single writing mode. Otherwise, if you're using multiple writing modes, try to use logical properties instead.
