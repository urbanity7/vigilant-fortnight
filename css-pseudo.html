Pseudo-classes

Pseudo-classes are used to style various states of an element. They are written as keywords that start with a colon (e.g., :hover), and are preceded by the type tag of the element of which you wish to target (in the case below, <a>):

a:hover {
}

Pseudo-classes tend to act as if you applied a class to a part of your document, enabling you to cut down on excess classes in your HTML.

Examples of pseudo-classes include:

  • :first-child. This is used to target the first child element.
  • :last-child. This is used to target the last child element.
  • :only-child. This is used to target an element without any siblings. It is equivalent to :first-child:last-child and :nth-child(1):nth-last-child(1), but with a lower specificity.
  • :invalid. This is used to represent any <form>, <fieldset>, or <input> elements whose contents fail to validate.

User-action or dynamic pseudo-classes only apply when a user interacts with the document in a specific way.

Examples of user-action pseudo-classes include:

  • :hover. As mentioned previously, this only applies if the user moves their pointer over an element, generally a link.
  • :focus. This only applies if the user focuses the element by clicking or using keyboard controls.

Pseudo-elements

Pseudo-elements select a certain part of an element rather than applying a class to existing elements. In other words, they behave as if you had added an entirely new HTML element into the markup. For instance, the ::first-line pseudo-element selects the first line of text in the element preceding the :: (in this case, the <p> element). Pseudo-elements are written similarly to pseudo-classes, except they are preceded by two colons rather than one:

p::first-line {
}

You might use the ::first-line pseudo-element as an alternative to enclosing your text in <span> elements. This is because the pseudo-element will always select the precise number of characters that can fit on the first line of text based on the screen's size. Since screen sizes change from device to device, what fits on the first line of text on one device might be different on a laptop than, say, an iPhone. If you were using the <span> option, you'd have to specify which text would always appear on the first line, as these tags would not change based on the size of the screen.

Combining pseudo-classes and pseudo-elements

If you wanted to make the first line of the first paragraph bold, you could chain the :first-child and ::first-line selectors together as such:

article p:first-child::first-line {
    font-size: 120%;
    font-weight: bold;
}

Note: the use of "article" here assumes the paragraphs that you're 
targeting are enclosed within a <article> tag within the <body>.

This declaration indicates that you want to select the first line of the first <p> element, which itself is located inside an <article> element.

Generating content with ::before and ::after

Special pseudo-elements, such as ::before and ::after, are used along with the content CSS property to insert content into your document. These might, among other things, be used to insert an icon after a given element:

Input:

CSS
.box::after {
    content: "➥";
}

HTML
<p class="box">Content in the box in my HTML page.</p>

Output:

Content in the box in my HTML page. ➥

Special pseudo-elements can also be used to insert an empty string, which can then be styled just like any element on the page:

Input:

CSS
.box::before {
    content: "";
    display: block;
    width: 100px;
    height: 100px;
    background-color: rebeccapurple;
    border: 1px solid black;
}   

HTML
<p class="box">Content in the box in my HTML page.</p>

Output:

Content in the box in my HTML page.

In this case, the empty string is set to display as display: block, so it will generate content accordingly based on the provided CSS. This content--the purple box--can then be styled like any other element using additional CSS.

In CSS, generated content refers to anything created using the ::before and ::after selectors and the content CSS property.
