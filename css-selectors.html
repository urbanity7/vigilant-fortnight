Selectors

Jump to type | class | ID | attribute | universal

Type selectors

These are used to select types of elements:

h1 {
}

Class selectors

Example:

.class-name {
}

To target a class on a particular element, use the type selector for the element you want to target and connect it with the .class (no space between):

p.example {
}

Keep in mind: you would need to make another selector if you wanted this rule to apply to other elements as well.

You can also pair classes with CSS selectors as such:

li.special {
    color: orange;
}

This syntax would apply the "special" class to all list items in your document:

li.special, span.special {
    color: orange;
}

If you were to do this, you would no longer be able to apply the "special" class to a <span> or another element 
by adding the class to it; you would have to add that element to the list of selectors.

Multiple classes may be applied to the same element. Simply separate the name of each class with no space between them when referring to them in the element (e.g., class="box big" would combine the box and big classes).

ID selectors

id selectors are similar to class selectors, except they can 1) only be used once per page and 2) cannot have multiple ID values attached to them in the same way that an element can have multiple class selectors:

#id-name {
}

ID selectors can select an element that has the id set on it, and you can precede the ID with a type selector to only target the element if both the element and ID match. See here for example code and potential usage.

Attribute selectors

These enable you to select elements based on the presence of a certain attribute on the element:

a[title] {
}

Attribute selectors can also be used to make a selection based on the presence of a certain value:

a[href="https://example.com"] {
}

Presence and value selectors

These types of attribute selectors enable the selection of an element based on the presence of an attribute alone (e.g., href), or on different matches against the value of the attribute.

Selector Example Description
[attr] a[title] Matches elements with an attr attribute (whose name is the value in the square brackets).
[attr=value] a[href="https://example.com"] Matches elements with an attr attribute whose value is exactly value--or the string inside the quotes.
[attr~=value] p[class~="special"] Matches elements with an attr attribute whose value is exactly value, or contains value in its (space separated) list of values.
[attr|=value] div[lang|="zh"] Matches elements with an attr attribute whose value is exactly value or begins with value immediately followed by a hyphen.

Example of potential usage (see here for list items being referenced):

  • By using li[class] we can match any list item with a class attribute. This matches all of the list items except the first one.
  • li[class="a"] matches a selector with a class of a, but not a selector with a class of a with another space-separated class as part of the value. It selects the second list item.
  • li[class~="a"] will match a class of a but also a value that contains the class of a as part of a whitespace-separated list. It selects the second and third list items.

Substring matching selectors

These selectors allow for more advanced matching of substrings inside the value of your attribute. For instance, if you had classes of box-warning and box-error and wanted to match everything that started with the string "box-", you could use [class^="box-"] to select them both (or [class|="box"] as described in section above).

Selector Example Description
[attr^=value] li[class^="box-"] Matches elements with an attr attribute whose value begins with value.
[attr$=value] li[class$="-box"] Matches elements with an attr attribute whose value ends with value.
[attr*=value] li[class*="box"] Matches elements with an attr attribute whose value contains value anywhere within the string.

(Aside: it may help to note that ^ and $ have long been used as anchors in so-called regular-expressions to mean begins with and ends with respectively.)

Example of potential usage (see here for list items being referenced):

  • li[class^="a"] matches any attribute value which starts with a, so matches the first two list items.
  • li[class$="a"] matches any attribute value which starts with a, so matches the first and third list items.
  • li[class*="a"] matches any attribute value where a appears anywhere in the string, so it matches all of our list items.

Universal selector

The universal selector selects everything in the document. If it is chained together with another element and descendant combinator, it will select everything in the parent element.

* {
}

It can be used to select any element which is the first-child of another element (in this case, the <article> element), like such:

article *:first-child{
    font-weight: bold;
}

Selector lists

When more than one selector uses the same CSS, you can combine the selectors into a selector list, separated by a comma:

p, .first-paragraph {
color: black;
font-size: 12px;
}

You can also write a selector list like this:

p, 
.first-paragraph {
color: black;
font-size: 12px;
}

Keep in mind: when using selector lists, if one part of the list is incorrect, then the CSS will not be applied to any component of the selector list.
