Selectors

Types of selectors

Type selectors are used to select types of elements:

h1 {
}

Class selectors

.class-name {
}

ID selectors

#id-name {
}

Attribute selectors enable you to select elements based on the presence of a certain attribute on the element:

a[title] {
}

Attribute selectors can also be used to make a selection based on the presence of a certain value:

a[href="https://example.com"] {
}

The universal selector selects everything in the document. If it is chained together with another element and descendant combinator, it will select everything in the parent element.

* {
}

Pseudo-classes and pseudo-elements

Pseudo-classes are used to style various states of an element:

a:hover {
}

Pseudo-elements select a certain part of an element rather than the entire element. For instance, the ::first-line pseudo-element selects the first line of text in the element preceding the :: (in this case, the <p> element):

p::first-line {
}

Combinators

Combinators combine other selectors to target groups of elements:

article > p {
}

Selector lists

When more than one selector uses the same CSS, you can combine the selectors into a selector list, separated by a comma:

p, .first-paragraph {
    color: black;
    font-size: 12px;
}

You can also write a selector list like this:

p, 
.first-paragraph {
    color: black;
    font-size: 12px;
}

Keep in mind: when using selector lists, if one part of the list is incorrect, then the CSS will not be applied to any component of the selector list.

Using classes

Classes are indicated by .class-name in CSS. You can pair classes with CSS selectors as such:

li.special {
    color: orange;
}

This syntax would apply the "special" class to all list items in your document. If you were to do this, you would no longer be able to apply the "special" class to a <span> or another element by adding the class to it; you would have to add that element to the list of selectors:

li.special, span.special {
    color: orange;
}

Some classes might be applied to many elements. In these cases, you don't want to have to keep editing your CSS every time something new needs to take on that style. Therefore, it is sometimes best to bypass the element and refer to the class, unless you know that you want to create some special rules for one element alone, or perhaps make sure they are not applied to other things.

Styling things based on their location in a document

Use a descendant combinator selector to select things based on their location:

li em {
    color: purple;
}

This selector will select any <em> element that is inside (a descendant of) an <li>. So in your example document, you should find that the <em> in the third list item is now purple, but the one inside the paragraph is unchanged.

Use an adjacent sibling combinator to style something that comes immediately after something else:

p + em {
    color: green;
}

note: this would target <em> elements that come after a <p> element closes; not within an open <p> element.

Styling things based on state

This is an example of how to target a link based on its state. Note: a:link refers to an unvisited link.

a:link {
    color: pink;
}

a:visited {
    color: green;
}

a:hover {
    color: red;
    text-decoration: none;
}

Combining selectors and combinators

You can combine multiple selectors and combinators together:

article p span {
    color: orange;
}
This would change a <span> located within a <p> inside an <article> to the color orange.

You can also combine multiple types together:

body h1 + p .special {
    color: yellow;
    background-color: black;
    padding: 5px;
}
This will style any element with a class of special, which is inside a <p>, which comes just after an <h1>, which is inside a <body>.

Three ways to add CSS

  • Using an external stylesheet, or the <link href> method
  • Using an internal stylesheet, or creating a <style> section within the HTML's <head>
  • Using inline styles by adding a style attribute to a given element
    • <h1 style="color: blue background-color: yellow; border: 1px solid black;">

Cascade vs specificity

  • Cascade: CSS will apply the rule that appears later in the stylesheet in instances where two or more rules are applied to the same element. Later styles will likewise replace earlier styles.
  • Specificity: styles applied to classes will override styles applied to elements because classes are considered to be more specific.

CSS functions

  • calc(): used to do simple math, e.g., calculate the given function
    • .h1 {
    calc(90% - 30px);
}
  • Transform functions
    • rotate(): used to rotate objects
      • .box {
    transform: rotate(0.8turn);
}
    • Other transform functions include skew(); matrix(); translate(); and scale().

@rules

@rules give instructions for how CSS should perform or behave. For example, you can use the @media rule to make a media query to change the background color of the page to blue when the viewport is wider than 30em:

body {
    background-color: pink;
}
      
@media (min-width: 30em) {
    body {
        background-color: blue;
    }
}
