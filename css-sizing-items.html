The natural or intrinsic size of things

HTML elements have a natural size that is set before they are affected by CSS. Consider an image: an image file contains sizing information, described as its intrinsic size. This size is determined by the image itself, and not by any formatting we might apply.

If you place an image on a page and do not change its height or width, either by using attributes on the <img> tag or by CSS, it will be displayed using its intrinsic size. This is the collapsed border on the element--there is no content to hold it open. However, if you add some text inside the empty <div> element, the border now contains that text because the height of the element is defined by the content. Therefore, the size of this <div> in the block dimension comes from the size of the content. Again, this is the intrinsic size of the element--its size defined by its content.

Using percentages

Percentages act like length units in many ways, and they can often be used interchangeably with lengths. When using a percentage, you need to be aware what it is a percentage of. In the case of a box inside another container, if you give the child box a percentage width it will be a percentage of the width of the parent container.

I have a percentage width.

This is because percentages resolve against the size of the containing block. With no percentage applied, our <div> would take up 100% of the available space, as it is a block level element. If we give it a percentage width, this becomes a percentage of the space it would normally fill.

Percentage margins and padding

If you set margins and padding as a percentage, you may notice some strange behavior. In the example below we have a box. We have given the inner box a margin of 10% and a padding of 10%. The padding a margin on the top and bottom of the box are the same size as the padding and margin on the left and right.

I have margin and padding set to 10% on all sides.

You might expect the percentage top and bottom margins, for example, to be a percentage of the element's height, and the percentage left and right margins to be a percentage of the element's width. This is not the case.

When you use margin and padding set in percentages, the value is calculated from the inline size of the containing block--or the width when working in a horizontal language. In our example, all of the margins and padding are 10% of the width. This means you can have equal-sized margins and padding all around the box. Keep this in mind if you use percentages.

Min- and max- sizes

In addition to giving things a fixed size, CSS can be used to give an element a minimum or a maximum size. If you have a box that might contain a variable amount of content, and you always want it to be at least a certain height, you could set the min-height property on it. The box will always be at least this height, but will then grow taller if there is more content than the box has space for at its minimum height. Both of the boxes below have a min-height of 75px. Because the box on the right has content that needs more room, it grows taller to accommodate this extra content.


These boxes both have a minimum height set. Because this box has more content in it, it needs more space than the assigned minimum height, and thus grows larger to accommodate this extra content.

This ability to set minimum and maximum sizes is therefore very useful when dealing with overflow.

A common use of max-width is to cause images to scale down if there is not enough space to display them at their intrinsic width, while also ensuring they don't become larger than this width. As an example, if you were to set width: 100% on an image, and its intrinsic width was smaller than its container, the image would be forced to stretch and become larger, therefore causing it to look pixelated. If you instead use max-width: 100%, and its intrinsic width is smaller than its container, the image will not be forced to stretch and become larger, allowing you to avoid pixelating the image.

In the example below, we have used the same image three times. The first image has been given width: 100% and is in a container which is larger than it. It therefore stretches to the container width. The second image has max-with: 75% set on it, and thus does not stretch to fill the container. The third box contains the same image again, also width max-width: 100% set; in this case, you can see how it has scaled down to fit the box.



This technique is used to make images responsive, so that when viewed on a smaller device they scale down appropriately. You should not use this technique to load huge images and then scale them down in the browser. Images should be appropriately sized to be no larger than they need to be for the largest size they are displayed in the design. Downloading overly large images will cause your site to slow, and can cost users more money if they are on a metered connection. You can learn more about responsive image techniques here.

Viewport units

The viewport is the visible area of your page in the browser you use to view a site. Like most things, it too has its own size. To size something relative to a user's viewport, use vw for viewport width, and vh for viewport height.

1vh is equal to 1% of the viewport height, while 1vw is equal to 1% of the viewport's width. You can use these units to size boxes as well as text. In the example below, we have a box which is sized as 20vh and 20vw. The box contains the letter "A," which has a font-size of 10vh.

If you change the vh and vw values, this will change the size of the box or font; changing the viewport size will also change their sizes because they are sized relative to the viewport. To see the example change when you change the viewport size, you will need to load the example in a new browser window that you can resize (as the embedded <iframe> that contains the example shown is above its viewport). Open the example, resize the browser window, and observe what happens to the size of the box and text.

Sizing things according to the viewport can be useful in your designs. For example, if you want a full-page hero section to show before the rest of your content, making that part of your page 100vh high will push the rest of the content below the viewport, meaning that it will only appear once the document is scrolled.
