Styling our table

Spacing and layout

Add the following CSS to adjust the spacing and layout of your table:

CSS
/* Spacing and layout */

table {
  table-layout: fixed;
  width: 100%;
  border-collapse: collapse;
  border: 3px solid purple;
}

thead th:nth-child(1) {
  width: 30%;
}

thead th:nth-child(2) {
  width: 20%;
}

thead th:nth-child(3) {
  width: 15%;
}

thead th:nth-child(4) {
  width: 35%;
}

th, 
td {
  padding: 20px;
}

Things to keep in mind:

  • A table-layout value of fixed is generally a good idea to set on your table, as it makes the table behave a bit more predictably by default. Normally, table columns tend to be sized according to how much content they contain, which produces some strange results. With table-layout: fixed, you can size your columns according to the width of their headings, and then deal with their content as appropriate. This is why we've selected the four different headings with the thead: th:nth-child(n) (nth-child) selector (see "Select the n-th child that is a <th> element in a sequence, inside a <thead> element") and given them set percentage widths. The entire column width follows the width of its heading, making for a nice way to size your table columns. This technique is discussed in more detail in Fixed Table Layouts. We've coupled this with a width of 100%, meaning that the table will fill any container it is put in, and be nicely responsive (although it would still need some work to get it looking good on narrow screen widths).

  • A border-collapse value of collapse is standard best practice for any table styling effort. By default, when you set borders on table elements, they will all have spacing between them, as the image below illustrates:

  • With border-collapse: collapse; set, the borders collapse down into one, which looks much better:

  • We've put a border around the whole table, which is needed because we'll be putting some borders round the table header and footer later on--it looks really odd and disjointed when you don't have a border round the whole outside of the table and end up with gaps.

  • We've set some padding on the <th> and <td> elements--this gives the data items space to breathe, making the table look a lot more legible.

Some simple typography

To use a font from Google Fonts, input the following HTML in your HTML head, above the existing <link> element:

HTML
<link
  href="https://fonts.googleapis.com/css?family=Rock+Salt"
  rel="stylesheet"
  type="text/css" />

You'll also need to add the following CSS to your style.css file, below the entry from the "Spacing and layout" section. Note the creation of a custom font-family declaration that includes the Google Font as well.

CSS
/* Typography */

html {
  font-family: "helvetica neue", helvetica, arial, sans-serif;
}

thead th, 
tfoot th {
  font-family: "Rock Salt", cursive;
}

th {
  letter-spacing: 2px;
}

td {
  letter-spacing: 1px;
}

tbody td {
  text-align: center;
}

tfoot th {
  text-align: right;
}

Things to keep in mind (note that these tips aren't exclusive to styling tables):

  • We've set a global sans-serif font stack; this is purely a stylistic choice. We've also set our custom font on headings inside the <thead> and <tfoot> elements for a more grunge look.
  • We've set some letter-spacing on the headings and cells to improve readability. This is likewise a primarily stylistic choice.
  • We've center-aligned the text in the table cells inside the <tbody> so that they line up with the headings. By default, cells are given a <text-align> value of left, and headings are given a value of center, but it generally looks better to have the alignments set the same for both. The default bold weight on the heading fonts is enough to differentiate their look, at least in this case.
  • We've right-aligned the heading inside the <tfoot> so that it is visually associated better with its data point.

Graphics and colors

To adjust your table's colors and graphics, you should first add the following CSS to your style.css file, again placing it beneath the previous section on typography:

CSS
/* Graphics and colors */

thead, 
tfoot {
  background: url(leopardskin.jpg);
  color: white;
  text-shadow: 1px 1px 1px black;
}

thead th, 
tfoot th, 
tfoot td {
  background: linear-gradient(
    to bottom,
    rgba(0, 0, 0, 0.1),
    rgba(0, 0, 0, 0.5)
  );
  border: 3px solid purple;
}

Things to keep in mind (note that these tips aren't exclusive to styling tables):

  • We've added a background-image to the <thead> and <tfoot>, and changed the color of all text inside the header and footer to white (as well as given it a text-shadow) so it is readable. It's a good idea to make sure your text contrasts well with your background for readability's sake.
  • We've added a linear gradient to the <th> and <td> elements inside the header and footer for a nice bit of texture, as well as given those elements a bright purple border. It is useful to have multiple nested elements available so you can layer styles on top of one another. Yes, we could have put both the background image and the linear gradient on the <thead> and <tfoot> elements using multiple background images, but we decided to do it separately for the benefit of older browsers that don't support multiple background images or linear gradients.

Zebra striping

This section is designed to show you how to implement zebra stripes--alternating rows of color in your table that make the data easier to read. Add the following CSS to you style.css file:

CSS
/* Zebra stripes */

tbody tr:nth-child(odd) {
  background-color: #ff33cc;
}

tbody tr:nth-child(even) {
  background-color: #e495e4;
}

tbody tr {
  background-image: url(noise.png);
}

table {
  background-color: #ff33cc;
}
  • Earlier on in this module, you saw the :nth-child selector being used to select specific child elements. It can also be given a formula as a parameter, so it will select a sequence of elements. The formula 2n+1 would select all the odd numbered children, and the formula 2n would select all the even numbered children. We've also used the odd and even keywords in our code, which do exactly the same things as these formulas. In this case, we are giving the odd and even rows different (lurid) colors.
  • We've added a repeating background tile to all the body rows, which is just a bit of noise (or to be more precise, a semi-transparent .png with a bit of visual distortion on it) to provide some texture.
  • We've given the entire table a solid background color so that browsers that don't support the :nth-child selector still have a background for their body rows.

Styling the caption

Lastly, to style the caption, add the following CSS to your style.css file:

CSS
/* Caption */

caption {
  font-family: "Rock Salt", cursive;
  padding: 20px;
  font-style: italic;
  caption-side: bottom;
  color: #666;
  text-align: right;
  letter-spacing: 1px;
}
  • The only thing of note here is the caption-side property, which has been given a value of bottom. This causes the caption to be positioned on the bottom of the table, which along with the other declarations, gives us this final looking table:

Table styling quick tips

To recap, here's a list of the most useful tips for styling tables:

  • Make your table markup as simple as possible, and keep things flexible (e.g., by using percentages) so the design is more responsive. It is also helpful to organize your CSS into sections so you can quickly find each portion of your table during the styling process.
  • Use table-layout: fixed to create a more predictable table layout that allows you to easily set column widths by setting width on their headings (<th>).
  • Use border-collapse: collapse to make table elements' borders collapse into each other, producing a neater and easier to control look.
  • Use <thead>, <tbody>, and <tfoot> to break up your table into logical chunks and provide extra places to apply CSS to, so it is easier to layer styles on top of one another if required.
  • Use zebra striping to make alternating rows easier to read.
  • Use text-align to line up your <th> and <td> text, to make things neater and easier to follow.
