Providing common styling to columns

Styling without <col>

HTML has a method of defining styling information for an entire column of data all in one place â€” the <col> and <colgroup> elements. These exist because it is inefficient having to use <td> or <th> in the column, or use a complex selector such as :nth-child.

  • Styling columns like this is limited to the following properties:
    • border
    • background
    • width
    • visibility
  • To set other properties, you have to either style every <td> or <th> in the column by setting a class with corresponding CSS rules, or use a complex selector such as :nth-child.

Styling with <col>

Instead of doing the above, we can input the information once, using the <col> element.

  • <col> elements are specified within a <colgroup> container below the opening <table> tag.

Input:

    <table>
        <colgroup>
            <col style="background-color: yellow" span="2" />
          </colgroup>
        <tr>
          <th>Data 1</th>
          <th>Data 2</th>
        </tr>
        <tr>
          <td>Calcutta</td>
          <td>Orange</td>
        </tr>
        <tr>
          <td>Robots</td>
          <td>Jazz</td>
        </tr>
      </table>

Adding structure with <thead>, <tfoot>, and <tbody>

Writing out the full element name in the case of <thead>, <tfoot>, and <tbody> allows you to give your HTML a structure and provide hooks from which you can utilize CSS. These elements do not increase accessibility.

<thead> This element must wrap the table's header--this is generally (but not always) the first row containing column headings. It comes directly after the <col>/<colgroup> if they're being used in your table.
<tfoot> This element must wrap the table's footer--for instance, this might be the table's final row containing the sum of the preceding rows a la an Excel document. It comes either right at the bottom of the table, or directly below the table header. In the case of the latter, the browser will still render it at the bottom of the table.
<tbody> This element must wrap the contents of the table that aren't contained within the table header or table footer. Depending on how you structure it, it will appear below the table header or table footer. Important: This element is always part of your HTML, even if not explicitly stated. It is still worth adding the explicit tag because it gives you more control over how the table is styled.
