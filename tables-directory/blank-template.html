Table cells should also be enclosed within <tr> tags. These tags are responsible for creating rows of cells. So, the hierarchy would be as follows:
<table>
<tr>
<th>
<td></td>
</th>
</tr>
</table>		 Note: <th> elements can also precede <tr> elements.
The height of table cells automatically resizes to fit the content in each row of cells. It is not based on the preceding row's height.

Providing common styling to columns

Styling without <col>

HTML has a method of defining styling information for an entire column of data all in one place â€” the <col> and <colgroup> elements. These exist because it is inefficient having to use <td> or <th> in the column, or use a complex selector such as :nth-child.

  • Styling columns like this is limited to the following properties:
    • border
    • background
    • width
    • visibility
  • To set other properties, you have to either style every <td> or <th> in the column by setting a class with corresponding CSS rules, or use a complex selector such as :nth-child.

Styling with <col>

Instead of doing the above, we can input the information once, using the <col> element.

  • <col> elements are specified within a <colgroup> container below the opening <table> tag.

Input:

    <table>
        <colgroup>
            <col style="background-color: yellow" span="2" />
          </colgroup>
        <tr>
          <th>Data 1</th>
          <th>Data 2</th>
        </tr>
        <tr>
          <td>Calcutta</td>
          <td>Orange</td>
        </tr>
        <tr>
          <td>Robots</td>
          <td>Jazz</td>
        </tr>
      </table>
